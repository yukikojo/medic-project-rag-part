# 多轮对话 Agent — 预设测试用例

> 用途：调试对话逻辑、验证 Agent 决策质量、发现追问策略问题
> 使用：复制每个 case 的初始症状 → `POST /api/rag/dialogue/start` → 按预期回答模拟患者 → 对比最终推荐

---

## 测试方法

```bash
# 1. 启动服务
uvicorn rag-db.src.api_server:app --host 0.0.0.0 --port 8000

# 2. 开始会话
curl -X POST http://localhost:8000/api/rag/dialogue/start \
  -H "Content-Type: application/json" \
  -d '{"initial_symptom": "模拟症状"}'

# 3. 继续对话（用返回的 session_id）
curl -X POST http://localhost:8000/api/rag/dialogue/continue \
  -H "Content-Type: application/json" \
  -d '{"session_id": "xxx", "patient_input": "患者回答"}'
```

## 测试用例

### Case 1：典型偏头痛（应推神经内科）

| 项 | 内容 |
|------|------|
| **症状** | 头痛三天了，右边太阳穴跳着疼 |
| **预期科室** | 神经内科 |
| **预期疾病** | 偏头痛 |

**模拟对话**：

| 轮次 | Agent 追问（预期方向） | 模拟回答 |
|------|----------------------|----------|
| 1 | 是否怕光/恶心？ | 有时候会恶心，看到强光更疼 |
| 2 | 发作频率/前兆？ | 隔一两天发作一次，发作前有时候眼前闪白光 |
| 3 | 什么能缓解？ | 睡一觉会好，吃布洛芬能缓解 |

**判断标准**：
- [ ] 追问方向不重复
- [ ] 最终推荐科室 = 神经内科
- [ ] 最终推荐疾病 = 偏头痛
- [ ] confidence > 0.65
- [ ] 3-4 轮内给出推荐

### Case 2：消化系统（应推消化内科）

| 项 | 内容 |
|------|------|
| **症状** | 肚子疼了三天，拉肚子 |
| **预期科室** | 消化内科 |
| **预期疾病** | 急性胃肠炎 |

**模拟对话**：

| 轮次 | Agent 追问 | 模拟回答 |
|------|-----------|----------|
| 1 | 疼痛位置/性质？ | 肚脐周围疼，一阵一阵的绞痛 |
| 2 | 大便性状/频率？ | 水样便，一天三四次 |
| 3 | 有没有发烧/吃了什么？ | 没发烧，昨天吃了海鲜大排档 |

**判断标准**：
- [ ] 追问到腹泻细节（频率/性状）
- [ ] 最终推荐科室 = 消化内科
- [ ] confidence > 0.65

### Case 3：呼吸系统（应推呼吸内科）

| 项 | 内容 |
|------|------|
| **症状** | 咳嗽咳了一周，有黄痰，有点发烧 |
| **预期科室** | 呼吸内科 |
| **预期疾病** | 急性支气管炎 / 肺炎 |

**模拟对话**：

| 轮次 | Agent 追问 | 模拟回答 |
|------|-----------|----------|
| 1 | 体温多少？有没有胸痛？ | 38度左右，胸口有点闷但不疼 |
| 2 | 痰什么颜色？有没有呼吸困难？ | 黄色的浓痰，没有喘不过气 |

**判断标准**：
- [ ] 追问到体温和痰的颜色
- [ ] 最终推荐科室 = 呼吸内科
- [ ] confidence > 0.65

### Case 4：紧急症状（应触发 emergency）

| 项 | 内容 |
|------|------|
| **症状** | 我爸突然剧烈胸痛，呼吸困难，出冷汗 |
| **预期动作** | action = emergency |
| **预期疾病** | 急性心肌梗死 |

**模拟对话**：

| 轮次 | Agent 行为 | 预期 |
|------|-----------|------|
| 1 | 立即返回 emergency，不追问 | action=emergency |

**判断标准**：
- [ ] 第一轮直接 emergency，不追问
- [ ] emergency_warning 包含"120"或"急诊"
- [ ] confidence > 0.9

---

### Case 5：模糊症状（应多轮追问）

| 项 | 内容 |
|------|------|
| **症状** | 最近有点不舒服 |
| **预期动作** | 追问具体哪里不舒服 |

**模拟对话**：

| 轮次 | Agent 追问 | 模拟回答 |
|------|-----------|----------|
| 1 | 具体哪里不舒服？ | 就是浑身没劲 |
| 2 | 持续多久了？有没有其他症状？ | 一个多星期了，有时候头晕 |
| 3 | 最近睡眠饮食怎么样？ | 睡得不好，吃饭没胃口 |

**判断标准**：
- [ ] 第一轮追问"哪里不舒服"
- [ ] 症状数 < 2 时强制追问（不推推荐）
- [ ] 追问方向逐步收窄
- [ ] 不出现重复追问

| 项 | 内容 |
|------|------|

**模拟对话**：

| 轮次 | Agent 追问 | 模拟回答 |

## 快速测试脚本

```bash
# 一次性跑所有 case 的首轮测试
for symptom in \
  "头痛三天右边太阳穴跳着疼" \
  "肚子疼拉肚子三天" \
  "咳嗽一周黄痰发烧" \
  "剧烈胸痛呼吸困难出冷汗" \
  "胳膊红疹很痒" \
  "眼睛红痒痛怕光" \
  "鼻子堵流浓鼻涕头疼" \
  "腰疼左边腿麻"
do
  echo "=== $symptom ==="
  curl -s -X POST http://localhost:8000/api/rag/dialogue/start \
    -H "Content-Type: application/json" \
    -d "{\"initial_symptom\": \"$symptom\"}" | python -m json.tool | grep -E '"action"|"department"|"disease"|"question"'
  echo ""
done
```
